## 009 HTTP Public API — слоты

### `POST /api/v1/slots` — многократные вызовы

**Статус:** открытый вопрос.

**Наблюдение:** после вызова `POST /api/v1/slots` его можно вызвать снова много раз (например, 150) — и столько же раз стартует workflow. Пока не ясно, насколько это желаемое поведение: нужны ли идемпотентность, дедупликация, лимиты, правило «один активный run на слот» и т.п.

**Дальше:** зафиксировать ожидаемую семантику в продукте/архитектуре и при необходимости обновить `specs/009-http-public-api/spec.md` и связанные контракты.
